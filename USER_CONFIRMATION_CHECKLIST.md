### 1. 环境变量与密钥配置

目前项目的`.env.local`包含占位符，需要你提供真实值。这些密钥**不会**被提交到代码库，只在本地和部署平台保管。

#### 需要提供的密钥：

| 变量名 | 说明 | 优先级 | 获取途径 |
|-------|------|-------|--------|
| `COZE_WORKLOAD_IDENTITY_API_KEY` | 豆包LLM认证API Key | 必需 | Coze工作台 → 身份认证管理 |
| `COZE_WORKLOAD_IDENTITY_CLIENT_SECRET` | 豆包LLM认证Secret | 必需 | 同上 |
| `COZE_SUPABASE_ANON_KEY` | Supabase 匿名密钥（服务端） | 必需 | Supabase Project → Settings → API Keys |
| `COZE_SUPABASE_SERVICE_ROLE_KEY` | Supabase Service Role密钥 | 必需 | 同上 |
| `STRIPE_SECRET_KEY` | Stripe生产密钥 | 必需 | Stripe Dashboard → API Keys |
| `STRIPE_WEBHOOK_SECRET` | Stripe Webhook签名密钥 | 必需 | Stripe Dashboard → Webhooks |
| `STRIPE_PRO_PRICE_ID` | Pro订阅价格ID | 必需 | Stripe Dashboard → Products |
| `STRIPE_UNLIMITED_PRICE_ID` | Unlimited订阅价格ID | 必需 | 同上 |
| `RUNPOD_API_KEY` | RunPod API Key (GPU图片生成) | 必需 | RunPod Dashboard → Settings |
| `RUNPOD_ENDPOINT_ID` | RunPod Endpoint ID | 必需 | RunPod Dashboard → Endpoints |
| `UPSTASH_REDIS_REST_URL` | Redis速率限制 | 可选 | Upstash → Database |
| `UPSTASH_REDIS_REST_TOKEN` | Redis Token | 可选 | 同上 |

#### 如何获取这些密钥：

**Coze (豆包)**:
1. 访问 https://console.coze.cn (中国) 或国际版
2. 进入项目 → 身份认证管理
3. 创建"工作负载身份"凭据
4. 复制 API Key 和 Client Secret

**Supabase**:
1. 访问 https://supabase.com → 登录你的项目
2. Settings → API Keys
3. 复制 `anon` key (公开)、`service_role` key (私密)

**Stripe**:
1. Stripe Dashboard → Developers → API Keys
2. 复制 Secret Key (以 `sk_` 开头)
3. Webhooks → 创建新webhook监听`payment_intent.succeeded`等事件
4. 复制Webhook Signing Secret

**RunPod**:
1. 登录 https://www.runpod.io
2. Dashboard → API Keys
3. 复制API Key
4. Endpoints → 记录你的Endpoint ID

   - [ ] 在欧盟/英国运营需要遵守GDPR，用户信息如何保管？

2. **法律合规**
   - [ ] 你咨询过律师关于"消费满$5000自动参加抽奖"的合法性吗？

   - [ ] 是否需要"隐私声明"和"奖励条款"页面（强制要求）？

3. **运营流程**
   - [ ] 实物奖励的采购渠道（官方Apple Store、认证商家、还是零售商）？
   - [ ] 如果收货地址无效或用户无法收货，退款/替代处理流程？
   - [ ] 是否需要用户签署税务表格（美国法律要求$600+奖励要报税）？
   - [ ] 每月最多发放几台iPhone？（建议：1-3台/月，以控制成本与法律风险）
